üéØ Binary Search Identification Guide

üîç When to Use Binary Search

Key Indicators:

  • Sorted Data: When the input array is sorted or can be sorted
  • Search Space Reduction: When you can eliminate half of the search space in each step
  • O(log n) Requirement: When the problem asks for logarithmic time complexity
  • Monotonic Function: When the answer follows a monotonic pattern (increasing or decreasing)

Common Problem Types:

  • Finding a specific value in a sorted array
  • Finding the first/last occurrence of an element
  • Finding the minimum/maximum value in a rotated array
  • Finding the peak element in a mountain array
  • Finding the median of two sorted arrays

üéØ Problem-Solving Strategy

Step-by-Step Approach:

  1. Identify the Search Space:
    • What are the possible values for the answer?
    • What are the minimum and maximum possible values?
  2. Define the Condition:
    • What condition determines if we should go left or right?
    • Is the condition monotonic (always increasing or decreasing)?
  3. Choose the Binary Search Type:
    • Standard binary search for exact matches
    • Leftmost binary search for first occurrence
    • Rightmost binary search for last occurrence

‚ö†Ô∏è Common Pitfalls & Solutions

Watch Out For:

  • Integer Overflow:
    // Wrong int mid = (left + right) / 2; // Correct int mid = left + (right - left) / 2;
  • Infinite Loops:
    // Wrong while (left < right) { if (condition) left = mid; else right = mid; } // Correct while (left < right) { if (condition) left = mid + 1; else right = mid; }
  • Off-by-One Errors:
    // Wrong return left; // Correct return left - 1; // or left, depending on the problem

üé® Binary Search Templates

1. Standard Binary Search (Exact Match):


int binarySearch(int[] arr, int target) {
    int left = 0, right = arr.length - 1;
    while (left <= right) {
        int mid = left + (right - left) / 2;
        if (arr[mid] == target) return mid;
        if (arr[mid] < target) left = mid + 1;
        else right = mid - 1;
    }
    return -1;
}

2. Leftmost Binary Search (First Occurrence):


int leftmostBinarySearch(int[] arr, int target) {
    int left = 0, right = arr.length;
    while (left < right) {
        int mid = left + (right - left) / 2;
        if (arr[mid] < target) left = mid + 1;
        else right = mid;
    }
    return left;
}

3. Rightmost Binary Search (Last Occurrence):


int rightmostBinarySearch(int[] arr, int target) {
    int left = 0, right = arr.length;
    while (left < right) {
        int mid = left + (right - left) / 2;
        if (arr[mid] <= target) left = mid + 1;
        else right = mid;
    }
    return left - 1;
}

üîë Key Decision Points

When to Use Each Template:

  • Standard Binary Search:
    • When you need to find an exact match
    • When the array is strictly sorted
    • When you need to return -1 if not found
  • Leftmost Binary Search:
    • When you need to find the first occurrence
    • When you need to find the insertion point
    • When you need to find the smallest element >= target
  • Rightmost Binary Search:
    • When you need to find the last occurrence
    • When you need to find the largest element <= target
    • When you need to find the number of elements <= target

1. Classic Binary Search (Exact Match)

Used when you want to find a specific target value in a sorted array.


int binarySearch(int[] arr, int target) {
    int left = 0, right = arr.length - 1;
    while (left <= right) {
        int mid = left + (right - left) / 2; // Prevents overflow
        if (arr[mid] == target)
            return mid; // Found target
        else if (arr[mid] < target)
            left = mid + 1; // Discard left half
        else
            right = mid - 1; // Discard right half
    }
    return -1; // Not found
}

2. Lower Bound (First Occurrence of Target or First ‚â• Target)

Useful for finding the left boundary (like insertion point).


int lowerBound(int[] arr, int target) {
    int left = 0, right = arr.length;
    while (left < right) {
        int mid = left + (right - left) / 2;
        if (arr[mid] < target)
            left = mid + 1; // Move right if mid is too small
        else
            right = mid; // Could be the answer, move left
    }
    return left; // Position to insert target
}

3. Upper Bound (First Element > Target)

Helps in finding strict boundaries.


int upperBound(int[] arr, int target) {
    int left = 0, right = arr.length;
    while (left < right) {
        int mid = left + (right - left) / 2;
        if (arr[mid] <= target)
            left = mid + 1;
        else
            right = mid;
    }
    return left; // First index where arr[i] > target
}

4. Binary Search on Answer (Search Space)

Used in problems where we don't search in array directly, but rather search for the minimum or maximum answer in a range.


int binarySearchAnswer(int low, int high) {
    while (low <= high) {
        int mid = low + (high - low) / 2;
        if (condition(mid))
            high = mid - 1; // Try to minimize
        else
            low = mid + 1;
    }
    return low; // Smallest value satisfying the condition
}

5. Binary Search in Rotated Sorted Array

üîç Pattern Overview

A rotated sorted array is a sorted array that has been rotated at some pivot point. For example:

Original: [1, 2, 3, 4, 5, 6, 7]
Rotated: [4, 5, 6, 7, 1, 2, 3]
Time Complexity: O(log n)
Space Complexity: O(1)

üéØ Key Insight

In a rotated sorted array, at least one half (left or right of mid) will always be sorted. This is the fundamental property we use to perform binary search.


int findInRotatedArray(int[] arr, int target) {
    int left = 0, right = arr.length - 1;
    
    while (left <= right) {
        int mid = left + (right - left) / 2;
        
        if (arr[mid] == target) return mid;
        
        // Check if left half is sorted
        if (arr[left] <= arr[mid]) {
            if (target >= arr[left] && target < arr[mid])
                right = mid - 1; // Search in left half
            else
                left = mid + 1; // Search in right half
        }
        // Right half must be sorted
        else {
            if (target > arr[mid] && target <= arr[right])
                left = mid + 1; // Search in right half
            else
                right = mid - 1; // Search in left half
        }
    }
    return -1;
}

‚ö†Ô∏è Common Pitfalls

  • Forgetting to check if one half is sorted before deciding where to search
  • Incorrect boundary conditions when checking if target is in sorted half
  • Not handling duplicate elements properly

‚úÖ Best Practices

  • Always check if one half is sorted before deciding search direction
  • Use inclusive bounds (left <= right) for exact match problems
  • Handle edge cases (empty array, single element) separately

6. Finding Minimum in Rotated Sorted Array

üîç Pattern Overview

Finding the minimum element in a rotated sorted array is equivalent to finding the pivot point where the rotation occurred.

Array: [4, 5, 6, 7, 1, 2, 3]
Minimum: 1 (at index 4)
Time Complexity: O(log n)
Space Complexity: O(1)

üéØ Key Insight

Compare mid with right element to determine which half contains the minimum. If arr[mid] > arr[right], minimum must be in right half.


int findMinInRotatedArray(int[] arr) {
    int left = 0, right = arr.length - 1;
    
    while (left < right) {
        int mid = left + (right - left) / 2;
        
        if (arr[mid] > arr[right])
            left = mid + 1; // Minimum is in right half
        else
            right = mid; // Minimum is in left half (including mid)
    }
    return arr[left];
}

‚ö†Ô∏è Common Pitfalls

  • Comparing with left element instead of right element
  • Using inclusive bounds when finding minimum
  • Not handling duplicate elements correctly

7. Binary Search in Mountain Array

üîç Pattern Overview

A mountain array (or bitonic array) first increases then decreases. The search requires finding the peak first, then searching in both parts.

Mountain Array: [1, 3, 5, 7, 6, 4, 2]
Peak: 7 (at index 3)
Time Complexity: O(log n)
Space Complexity: O(1)

üéØ Key Insight

The search process is two-phase: first find the peak, then perform binary search in both increasing and decreasing parts.


int findInMountainArray(int[] arr, int target) {
    // First find the peak
    int peak = findPeak(arr);
    
    // Search in increasing part
    int leftResult = binarySearch(arr, target, 0, peak, true);
    if (leftResult != -1) return leftResult;
    
    // Search in decreasing part
    return binarySearch(arr, target, peak + 1, arr.length - 1, false);
}

int findPeak(int[] arr) {
    int left = 0, right = arr.length - 1;
    
    while (left < right) {
        int mid = left + (right - left) / 2;
        
        if (arr[mid] < arr[mid + 1])
            left = mid + 1; // Peak is in right half
        else
            right = mid; // Peak is in left half (including mid)
    }
    return left;
}

int binarySearch(int[] arr, int target, int left, int right, boolean isAscending) {
    while (left <= right) {
        int mid = left + (right - left) / 2;
        
        if (arr[mid] == target) return mid;
        
        if (isAscending) {
            if (arr[mid] < target)
                left = mid + 1;
            else
                right = mid - 1;
        } else {
            if (arr[mid] < target)
                right = mid - 1;
            else
                left = mid + 1;
        }
    }
    return -1;
}

‚ö†Ô∏è Common Pitfalls

  • Not handling the peak element correctly
  • Using same comparison logic for both increasing and decreasing parts
  • Incorrect boundary conditions when finding peak

‚úÖ Best Practices

  • Always find peak first, then search in both parts
  • Use different comparison logic for increasing and decreasing parts
  • Handle edge cases (single element, two elements) separately

8. Finding Peak in Mountain Array

üîç Pattern Overview

Finding the peak in a mountain array is a crucial first step for many mountain array problems.

Array: [1, 3, 5, 7, 6, 4, 2]
Peak: 7 (at index 3)
Time Complexity: O(log n)
Space Complexity: O(1)

üéØ Key Insight

Compare mid with mid+1 to determine which half contains the peak. If arr[mid] < arr[mid+1], peak is in right half.


int findPeakInMountainArray(int[] arr) {
    int left = 0, right = arr.length - 1;
    
    while (left < right) {
        int mid = left + (right - left) / 2;
        
        if (arr[mid] < arr[mid + 1])
            left = mid + 1; // Peak is in right half
        else
            right = mid; // Peak is in left half (including mid)
    }
    return left;
}

‚ö†Ô∏è Common Pitfalls

  • Comparing with previous element instead of next element
  • Using inclusive bounds when finding peak
  • Not handling edge cases properly

üß† Additional Tips for Rotated & Mountain Arrays

üîç Pattern-Specific Tips

  • For Rotated Arrays:
    • Always check if one half is sorted before deciding where to search
    • Be careful with duplicate elements - they can make it hard to determine which half is sorted
    • For finding minimum, compare with right element instead of left
    • Use inclusive bounds (left <= right) for exact match problems
  • For Mountain Arrays:
    • First find the peak, then search in both parts
    • Remember to use different comparison logic for increasing and decreasing parts
    • For peak finding, comparing with next element is more reliable than comparing with previous
    • Handle edge cases (single element, two elements) separately

üéØ General Tips

  • Always verify your binary search conditions
  • Test with edge cases (empty array, single element, two elements)
  • Consider using inclusive or exclusive bounds based on the problem
  • Be careful with integer overflow when calculating mid
